# 第五章：向量存储（VectorStore）

## 什么是向量数据库？

向量数据库专门用于**存储和检索向量**。传统数据库按精确值查询（WHERE id = 5），向量数据库按"相似度"查询（找到最相似的 K 个向量）。

```
查询向量 [0.1, -0.4, 0.8, ...]
       ↓
在 100 万个向量中找到「最相似的 5 个」
       ↓
返回: [(id=42, similarity=0.94), (id=7, similarity=0.91), ...]
```

## Chroma 简介

Chroma 是最流行的**轻量级向量数据库**，特点：

- **纯 Python**：安装简单，pip install chromadb
- **本地存储**：数据存在本地文件（SQLite），无需启动服务
- **功能完整**：支持元数据过滤、HNSW 索引、集合管理
- **OpenAI 兼容**：和我们的 Embedding API 无缝配合

## Chroma 的核心概念

```
Chroma Client
    ↓
Collection（集合）← 类似"表"
    ↓
Chunk（块）← 包含：id, embedding, document(text), metadata
```

## 核心代码实现

### 5.1 基础：创建 Collection 和添加向量

```python
# src/vectorstore.py

import chromadb
from chromadb.config import Settings
from uuid import uuid4

class VectorStore:
    """Chroma 向量数据库封装。"""

    def __init__(self, persist_directory: str = "data/chroma_db"):
        self.client = chromadb.PersistentClient(
            path=persist_directory,
            settings=Settings(anonymized_telemetry=False),  # 关闭遥测
        )
        self.collection = None

    def create_collection(
        self,
        name: str = "documents",
        embedding_dim: int = 1024,
    ):
        """创建集合（如果已存在则直接获取）。"""
        self.collection = self.client.get_or_create_collection(
            name=name,
            metadata={"hnsw:space": "cosine"},  # cosine = 余弦距离
        )
        self._embedding_dim = embedding_dim

    def add_chunks(self, chunks: list[dict]):
        """
        将 chunks 添加到集合。
        每条记录包含：id, embedding, document, metadata
        """
        if self.collection is None:
            raise ValueError("Collection 未创建，请先调用 create_collection()")

        ids = [str(uuid4()) for _ in chunks]
        embeddings = [chunk["embedding"] for chunk in chunks]
        documents = [chunk["text"] for chunk in chunks]
        metadatas = [chunk.get("metadata", {}) for chunk in chunks]

        self.collection.add(
            ids=ids,
            embeddings=embeddings,
            documents=documents,
            metadatas=metadatas,
        )

        return ids  # 返回的 id 可用于后续查询
```

### 5.2 检索：向量相似度查询

```python
    def query(
        self,
        query_embedding: list[float],
        n_results: int = 5,
        where_filter: dict = None,  # 元数据过滤，如 {"source": "doc1.pdf"}
    ) -> dict:
        """
        检索最相似的文档块。
        返回: {
            "ids": [[...]],         # 匹配的 ID 列表
            "documents": [[...]],   # 匹配的文本
            "metadatas": [[...]],   # 匹配的元数据
            "distances": [[...]],   # 距离（越小越相似）
        }
        """
        results = self.collection.query(
            query_embeddings=[query_embedding],
            n_results=n_results,
            where=where_filter,
            include=["documents", "metadatas", "distances"],
        )
        return results
```

### 5.3 元数据过滤

这是 RAG 中**按文档筛选**的关键功能：

```python
# 例子：只查询来自 "report.pdf" 的内容
results = vectorstore.query(
    query_embedding=user_embedding,
    n_results=5,
    where_filter={"source": "report.pdf"},  # 只查这个文档
)
```

Chroma 支持的过滤操作符：
```python
{"source": {"$eq": "report.pdf"}}           # 等于
{"chunk_index": {"$gte": 10}}              # 大于等于
{"$and": [{"source": "a.pdf"}, {"chunk_index": {"$lt": 50}}]}  # AND 组合
```

### 5.4 删除文档

删除时需要知道 Chroma 分配的 ID，不能按文件名删除。所以元数据里必须存 `source`：

```python
    def delete_by_metadata(self, where_filter: dict):
        """删除所有满足条件的记录。"""
        # 先查到所有满足条件的 ID
        results = self.collection.get(where=where_filter)
        if results["ids"]:
            self.collection.delete(ids=results["ids"])

    def delete_collection(self, name: str = "documents"):
        """删除整个集合。"""
        self.client.delete_collection(name=name)
        if self.collection is not None:
            self.collection = None
```

### 5.5 HNSW 索引（重要！）

HNSW（Hierarchical Navigable Small World）是 Chroma 默认的向量索引算法，直接影响检索速度和质量。

| 参数 | 默认值 | 调整建议 |
|------|--------|---------|
| `hnsw:space` | `"cosine"` | `"l2"`（欧氏距离）或 `"ip"`（内积） |
| `hnsw:ef_construction` | `100` | 越高越精确，但构建越慢 |
| `hnsw:ef_search` | `100` | 越高越精确，检索越慢 |
| `hnsw:max_neighbors` | `16` | 邻居数，影响召回率 |

```python
# 创建时配置 HNSW
self.collection = self.client.get_or_create_collection(
    name=name,
    metadata={
        "hnsw:space": "cosine",
        "hnsw:ef_construction": 200,  # 从 100 提高到 200，更精确
    },
)

# 动态调整搜索精度（不需要重建索引）
self.collection.config_for_ef_search(n=ef_search)  # ef_search ↑ = 更准但更慢
```

### 5.6 增量 Upsert（重要！）

朴素实现：每次文档变了，删除整个 collection → 重建。

问题：10000 个文档，每次改 1 个文档就要重建全部 → **不可接受**。

增量 Upsert 方案：
```
文档来了
    ↓
计算 content_hash（文本内容的哈希）
    ↓
查 Chroma 中已有的 hash
    ↓
hash 相同 → 跳过（未变）
hash 不同 → upsert（变了）
    ↓
Chroma 中有但新文档没有 → 删除（被删了）
```

```python
    def upsert_document(
        self,
        chunks: list[dict],
        source: str,
        embedder,  # 用于计算新 chunk 的 embedding
    ):
        """
        增量更新：只处理新增/变化/删除的块，复用未变化的 embedding。
        """
        import hashlib

        # 1. 计算新 chunks 的 hash
        for chunk in chunks:
            content = chunk["text"]
            chunk["content_hash"] = hashlib.sha256(content.encode()).hexdigest()[:16]
            chunk["metadata"]["source"] = source

        # 2. 获取集合中已有的该文档的 chunks
        existing = self.collection.get(
            where={"source": source},
            include=["documents", "metadatas"],
        )

        existing_hashes = set()
        existing_ids_map = {}  # hash → id

        if existing["ids"]:
            for i, doc in enumerate(existing["documents"]):
                h = existing["metadatas"][i].get("content_hash", "")
                existing_hashes.add(h)
                existing_ids_map[h] = existing["ids"][i]

        # 3. 分类：新 chunks
        new_hashes = set(c["content_hash"] for c in chunks)

        # 新增/更新
        to_upsert = []
        to_upsert_ids = []
        for chunk in chunks:
            h = chunk["content_hash"]
            if h not in existing_hashes:
                to_upsert.append(chunk)
                to_upsert_ids.append(existing_ids_map.get(h, str(uuid4())))

        # 删除（Chroma 有但新文档没有的）
        to_delete_hashes = existing_hashes - new_hashes
        to_delete_ids = [existing_ids_map[h] for h in to_delete_hashes if h in existing_ids_map]

        # 4. 执行
        if to_delete_ids:
            self.collection.delete(ids=to_delete_ids)

        if to_upsert:
            # 对新 chunks 计算 embedding（变化了的才算）
            texts = [c["text"] for c in to_upsert]
            embeddings, _ = embedder.embed_batch(texts)
            for i, chunk in enumerate(to_upsert):
                chunk["embedding"] = embeddings[i]

            ids = [existing_ids_map.get(c["content_hash"], str(uuid4())) for c in to_upsert]
            self.collection.upsert(
                ids=ids,
                embeddings=[c["embedding"] for c in to_upsert],
                documents=[c["text"] for c in to_upsert],
                metadatas=[c["metadata"] for c in to_upsert],
            )
```

**Upsert 的效果**：
- 10000 个文档，改了 1 个 → 重新向量化 1 个，而不是 10000 个
- 未变化的 chunk 的 embedding 直接复用，节省 API 调用

## 完整使用流程

```python
# 1. 初始化
vs = VectorStore("data/chroma_db")
vs.create_collection("my_docs", embedding_dim=1024)

# 2. 添加文档
chunks = [
    {"text": "RAG 是检索增强生成...", "embedding": [0.1, ...], "metadata": {"source": "intro.pdf"}},
    {"text": "向量数据库存储向量...", "embedding": [0.2, ...], "metadata": {"source": "intro.pdf"}},
]
vs.add_chunks(chunks)

# 3. 检索
results = vs.query(
    query_embedding=user_question_embedding,
    n_results=3,
    where_filter={"source": "intro.pdf"},
)
print(results["documents"][0])  # 最相似的文档

# 4. 删除
vs.delete_by_metadata({"source": "intro.pdf"})
```
