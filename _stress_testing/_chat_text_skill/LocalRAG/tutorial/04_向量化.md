# 第四章：向量化（Embedding）

## 什么是 Embedding？

Embedding（嵌入）是把**文本变成一串数字**的技术。

```
"什么是 RAG"  →  [0.123, -0.456, 0.789, ..., 0.001]  (1024 维向量)
"RAG 是检索增强生成"  →  [0.234, -0.567, 0.890, ..., 0.002]
```

为什么这样做？因为**语义相近的文本，向量也相近**：

```
"什么是 RAG"
        ↕ 语义很近
"RAG 的定义"
        ↕ 语义很远
"今晚吃什么"
```

我们可以用**向量距离**（余弦相似度、欧氏距离等）来衡量文本之间的语义相似度。

## Embedding 模型

常用模型：

| 模型 | 维度 | 特点 |
|------|------|------|
| OpenAI `text-embedding-3-small` | 1536 | 效果好，收费 |
| BGE (`bge-large-zh-v1.5`) | 1024 | 中文优秀，开源免费 |
| `all-MiniLM-L6-v2` | 384 | 轻量快速，英文为主 |

我们使用 OpenAI 兼容 API，所以 BGE 和 OpenAI 模型可以互换。

## Embedding API 接口

大多数 Embedding API 都遵循 OpenAI 的格式：

```python
# 请求
POST /v1/embeddings
{
  "model": "bge-large-zh-v1.5",
  "input": ["要向量化的一句话", "另一句话"]
}

# 响应
{
  "data": [
    {"embedding": [0.123, ...], "index": 0},
    {"embedding": [0.456, ...], "index": 1}
  ],
  "usage": {"prompt_tokens": 10, "total_tokens": 10}
}
```

## 完整代码实现

### 4.1 Embedding 缓存

Embedding 是"相同文本 → 相同向量"，相同内容不要重复调用 API。

```python
# src/embedder_api.py（部分）

from cachetools import TTLCache
import hashlib

class EmbeddingCache:
    """LRU + TTL 缓存，线程安全。"""

    def __init__(self, max_size: int = 10000, ttl: int = 3600):
        # TTLCache: 超过 TTL 的自动过期
        self._cache = TTLCache(maxsize=max_size, ttl=ttl)
        self._stats = {"hits": 0, "misses": 0}

    def _hash(self, text: str) -> str:
        """用 SHA-256 生成稳定 key。"""
        return hashlib.sha256(text.encode()).hexdigest()[:32]

    def get(self, text: str):
        key = self._hash(text)
        result = self._cache.get(key)
        if result is not None:
            self._stats["hits"] += 1
        else:
            self._stats["misses"] += 1
        return result

    def set(self, text: str, embedding: list):
        key = self._hash(text)
        self._cache[key] = embedding
```

`max_size=10000, ttl=3600` 的意思是：最多缓存 10000 条，**每条 1 小时后自动过期**。

### 4.2 重试逻辑（防网络抖动）

Embedding API 可能因为网络问题暂时失败，用**指数退避**重试：

```python
# src/embedder_api.py（部分）

import tenacity
import time

DEFAULT_MAX_ATTEMPTS = 3
DEFAULT_INITIAL_BACKOFF = 1.0  # 首次失败等 1 秒
DEFAULT_BACKOFF_FACTOR = 2.0    # 每次失败后等待时间翻倍
DEFAULT_MAX_BACKOFF = 10.0      # 最多等 10 秒

def _is_retryable(exception):
    """判断这个错误是否值得重试。"""
    from openai import RateLimitError, APITimeoutError, APIError
    if isinstance(exception, (RateLimitError, APITimeoutError)):
        return True
    if isinstance(exception, APIError):
        return 500 <= exception.status_code < 600  # 服务端错误才重试
    return "timeout" in str(exception).lower()

class EmbedderAPIClient:
    def __init__(self, api_base: str, api_key: str, model: str):
        self.client = OpenAI(base_url=api_base, api_key=api_key)
        self.model = model
        self.cache = EmbeddingCache()

    def _call_api(self, texts: list[str]) -> list[list[float]]:
        """调用 API，自动重试。"""

        @tenacity.retry(
            retry=tenacity.retry_if_exception(_is_retryable),
            wait=tenacity.wait_exponential(
                multiplier=1.0,
                exp_base=2.0,
                max=10.0,
            ),
            stop=tenacity.stop_after_attempt(3),
            reraise=True,
        )
        def _call():
            response = self.client.embeddings.create(
                model=self.model,
                input=texts,
            )
            return [item.embedding for item in response.data]

        return _call()
```

**重试流程（3次尝试）：**
```
第1次: 失败 → 等 1s  → 第2次
第2次: 失败 → 等 2s  → 第3次
第3次: 失败 → 抛异常
```

### 4.3 批处理（节省 API 调用）

Embedding API 按**次计费**，一次发 100 条比发 100 次单条便宜得多且快得多。

```python
    def embed_batch(
        self,
        texts: list[str],
        use_cache: bool = True,
        skip_on_error: bool = False,
    ) -> tuple[list[list[float]], list[int]]:
        """
        批量向量化，自动分 sub_batch。
        返回 (embeddings, failed_indices)
        """
        all_embeddings: list[Optional[list[float]]] = [None] * len(texts)
        failed_indices: list[int] = []

        # 1. 命中缓存的直接返回
        uncached_indices = []
        if use_cache:
            for i, text in enumerate(texts):
                cached = self.cache.get(text)
                if cached is not None:
                    all_embeddings[i] = cached
                else:
                    uncached_indices.append(i)
        else:
            uncached_indices = list(range(len(texts)))

        if not uncached_indices:
            return all_embeddings, failed_indices

        # 2. 分 sub_batch 调用 API（每次最多 16 条）
        sub_batch_size = 16
        for start in range(0, len(uncached_indices), sub_batch_size):
            sub = uncached_indices[start:start + sub_batch_size]
            sub_texts = [texts[i] for i in sub]

            try:
                sub_embeddings = self._call_api(sub_texts)
                for idx, emb in zip(sub, sub_embeddings):
                    all_embeddings[idx] = emb
                    if use_cache:
                        self.cache.set(texts[idx], emb)
            except Exception as e:
                if skip_on_error:
                    for idx in sub:
                        failed_indices.append(idx)
                        all_embeddings[idx] = [0.0] * 1536  # 占位符
                else:
                    failed_indices.extend(sub)
                    raise  # 向上传播

        return all_embeddings, failed_indices
```

**为什么要分 sub_batch？**
- API 有单次最大输入限制（通常是 2048 tokens 或 8192 tokens）
- 一次发太多文本，API 会报 400 错误
- sub_batch_size=16 是经验值，兼容大多数 API

## 向量维度的重要性

**每条向量必须维度一致！** Chroma 会检查：
```
向量A: [0.1, 0.2, 0.3]        (3维)
向量B: [0.4, 0.5, 0.6, 0.7]  (4维) ← 报错！维度不一致
```

如果你换了 Embedding 模型（比如从 1536 维换成 1024 维），**旧的向量数据库必须重建**，因为维度不兼容。
