# 第十章：部署与运维

## 10.1 本地运行

最简单的方式，直接运行：

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动 LM Studio（确保 LLM 服务在运行）

# 3. 启动 Streamlit
streamlit run src/app.py --server.port 8501 --server.headless true
```

打开浏览器 `http://localhost:8501` 即可使用。

## 10.2 使用 run.py 启动脚本

项目根目录的 `run.py` 提供了统一的启动入口：

```python
#!/usr/bin/env python3
"""Runner script for Local RAG application."""
import sys
from pathlib import Path
project_root = Path(__file__).parent
sys.path.insert(0, str(project_root / "src"))
from app import main
if __name__ == "__main__":
    main()
```

```bash
python run.py
# 或
streamlit run run.py --server.port 8501
```

## 10.6 Systemd 服务（Linux 服务器）

如果部署在 Linux 服务器上，用 systemd 管理进程：

```ini
# /etc/systemd/system/localrag.service

[Unit]
Description=LocalRAG Streamlit Application
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/LocalRAG
ExecStart=/opt/LocalRAG/.venv/bin/streamlit run src/app.py \
    --server.port 8501 \
    --server.headless true \
    --server.address 0.0.0.0
Restart=always
RestartSec=5
Environment="ENV=production"

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable localrag
sudo systemctl start localrag
sudo systemctl status localrag  # 查看状态
```

查看日志：
```bash
journalctl -u localrag -f  # 实时查看
```

## 10.7 Nginx 反向代理

如果需要域名 + HTTPS：

```nginx
# /etc/nginx/sites-available/localrag

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 300;
    }
}
```

## 10.8 Docker 部署

```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8501

CMD ["streamlit", "run", "src/app.py", "--server.port", "8501", "--server.address", "0.0.0.0"]
```

```bash
# 构建镜像
docker build -t localrag:latest .

# 运行容器
docker run -d \
  --name localrag \
  -p 8501:8501 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/logs:/app/logs \
  localrag:latest
```

## 10.9 常见问题排查

| 问题 | 排查方法 |
|------|---------|
| LM Studio 连不上 | 检查 `http://localhost:1234/v1` 是否可访问 |
| Embedding 维度错误 | 删除 `data/chroma_db/` 重建 |
| Streamlit 页面空白 | 查看 `logs/app.log`，检查端口占用 |
| 检索返回空结果 | 确认文档已索引，检查向量数据库文件权限 |

## 10.10 运维检查清单

```
日常检查：

定期检查：

□ requirements.txt 是否有安全更新
```

# 总结

## RAG 十大核心知识点

```
┌─────────────────────────────────────────────────────────────┐
│  1. 文档加载     │ loader.py    │ 把 PDF/TXT/MD 读入 Python    │
│  2. 分块策略     │ chunkers/    │ 5 种策略，按场景选           │
│  3. 向量化       │ embedder_api │ API + 缓存 + 批处理 + 重试    │
│  4. 向量存储     │ vectorstore  │ Chroma + HNSW + 增量 Upsert   │
│  5. 混合检索     │ retriever    │ 向量 + BM25 + RRF 融合        │
│  6. MMR 多样性   │ retriever    │ lambda 控制相关/多样平衡       │
│  7. 重排序       │ retriever    │ CrossEncoder 4 模式           │
│  8. 生成         │ llm_api      │ Prompt 构建 + 流式输出 + 重试  │
│  9. 对话历史     │ history_mgr  │ JSON 持久化 + 反馈收集         │
│ 10. 可观测性     │ observability│ structlog + Prometheus + 追踪  │
└─────────────────────────────────────────────────────────────┘
```

## 快速参考：各模块依赖关系

```
loader.py        ← 依赖 pypdf
    ↓
chunkers/       ← 依赖 embedder（semantic/llm 需要）
    ↓
embedder_api.py ← 依赖 OpenAI API
    ↓
vectorstore.py  ← 依赖 Chroma
    ↓
retriever.py    ← 依赖 vectorstore + embedder + BM25
    ↓
llm_api.py     ← 依赖 OpenAI API
    ↓
pipeline.py     ← 整合 retriever + llm_api
    ↓
app.py          ← 整合 pipeline + history_manager + observability
```
