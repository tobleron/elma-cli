# 第六章：检索原理（Retriever）

## 检索要解决什么问题？

用户提问 "RAG 的分块策略有哪些"，我们需要：
1. 把问题变成向量
2. 在向量数据库中找到**最相关**的文档块

但仅靠向量检索（纯语义）有以下问题：

| 问题 | 例子 | 原因 |
|------|------|------|
| **关键词不匹配** | 问"分块策略"，但文档写的是"chunker" | 同义词，embedding 可能学到了 |
| **专有名词** | 问"HNSW"，但文档里是"HNSW 算法" | 专有名词有时语义偏移 |
| **数值精确匹配** | 问"2024年"，文档里也是"2024年" | 数字类查询向量效果差 |

解决方案：**混合检索** = 向量检索 + 关键词检索（BM25）。

## 6.1 BM25（经典关键词检索）

BM25 是搜索引擎 decades 验证过的算法，和向量无关，纯靠词频统计：

```python
# src/retriever.py（部分）
from rank_bm25 import BM25Plus

class BM25Retriever:
    """纯关键词检索，不依赖向量。"""

    def __init__(self, k1: float = 1.5, b: float = 0.75):
        self.k1 = k1  # 词频饱和参数
        self.b = b    # 文档长度归一化参数
        self._index = None
        self._doc_texts = []

    def index(self, documents: list[str]):
        """构建 BM25 索引。"""
        self._doc_texts = documents
        tokenized = [doc.lower().split() for doc in documents]
        self._index = BM25Plus(tokenized, k1=self.k1, b=self.b)

    def retrieve(self, query: str, top_k: int = 10) -> list[tuple[int, float]]:
        """
        检索，返回 [(doc_index, score), ...]
        score 越高越相关。
        """
        tokenized_query = query.lower().split()
        scores = self._index.get_scores(tokenized_query)

        # 按分数排序，取 top_k
        doc_scores = sorted(enumerate(scores), key=lambda x: x[1], reverse=True)
        return doc_scores[:top_k]
```

BM25 的核心思想：**一个词在文档中出现越多越相关，但出现太多会饱和**。

## 6.2 混合检索（Hybrid Search）

把向量检索和 BM25 的分数用 **RRF（Reciprocal Rank Fusion）** 融合：

```python
def reciprocal_rank_fusion(
    vector_results: list[tuple[str, float]],   # [(doc_id, score), ...]
    bm25_results: list[tuple[int, float]],      # [(doc_index, score), ...]
    rrf_k: int = 60,
) -> list[tuple[str, float]]:
    """
    RRF 融合算法。
    rrf_k: 越大越偏向 BM25 的排名结果
    """
    scores = {}

    # 向量检索结果打分
    for rank, (doc_id, _) in enumerate(vector_results):
        scores[doc_id] = scores.get(doc_id, 0) + 1 / (rrf_k + rank + 1)

    # BM25 结果打分
    for rank, (doc_idx, _) in enumerate(bm25_results):
        # 这里用 index 代替 doc_id，实际代码需要映射
        doc_id = str(doc_idx)
        scores[doc_id] = scores.get(doc_id, 0) + 1 / (rrf_k + rank + 1)

    # 按总分排序
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

**为什么用 RRF 而不是直接加权平均？**
- 加权平均需要调参（向量权重 + BM25 权重）
- RRF 只用一个参数 `rrf_k`，不依赖分数的绝对值，更鲁棒

## 6.3 MMR（最大边际相关性）

问题：向量检索倾向返回**语义相似但内容重复**的文档。

```
用户问：RAG 的分块策略
向量检索结果：
  1. "固定大小分块：按字符数切分..." (score=0.92)
  2. "固定大小分块：overlap 参数..." (score=0.91)  ← 内容重叠！
  3. "递归分块：按分隔符..."        (score=0.88)
```

MMR 的解决思路：**结果要多样，不能全是同类型的**。

```python
def mmr_retrieve(
    query_embedding: list[float],
    candidate_docs: list[dict],    # 候选文档，含 embedding
    lambda_param: float = 0.3,     # 0=纯相关性，1=纯多样性
    top_k: int = 5,
) -> list[dict]:
    """
    MMR: 在相关性和多样性之间取平衡。
    lambda_param 越大 → 越追求多样性。
    """
    selected = []
    remaining = candidate_docs.copy()

    for _ in range(top_k):
        best_score = -float("inf")
        best_doc = None

        for doc in remaining:
            # 相关性分数
            relevance = cosine_similarity(query_embedding, doc["embedding"])

            # 多样性分数（与已选文档的最大相似度，越小越多样）
            if selected:
                max_sim_to_selected = max(
                    cosine_similarity(doc["embedding"], s["embedding"])
                    for s in selected
                )
            else:
                max_sim_to_selected = 0

            # MMR 公式
            mmr_score = lambda_param * relevance - (1 - lambda_param) * max_sim_to_selected

            if mmr_score > best_score:
                best_score = mmr_score
                best_doc = doc

        if best_doc:
            selected.append(best_doc)
            remaining.remove(best_doc)

    return selected
```

**MMR 公式解释：**
```
MMR = λ × relevance - (1-λ) × max_similarity_to_selected

λ=0.3 时：
  → 70% 权重给相关性，30% 权重给多样性
  → 结果既相关，又不重复
```

## 6.4 重排序（Reranking）

两阶段检索的最后一环：

```
阶段1（粗排）：向量检索 → 取 top 20
         ↓
阶段2（精排）：CrossEncoder 重新排序 → 取 top 5
```

为什么需要两阶段？
- 向量数据库擅长在大规模数据中快速找到"差不多相关"的候选集
- CrossEncoder 精度更高，但计算慢（需要逐个对比 query 和 doc）

### CrossEncoder 的原理

向量检索：一个文本 → 一个向量 → 预计算好存储
CrossEncoder：query + doc → 一起进模型 → 直接输出相似度分数

```
向量检索（bi-encoder）:
  "RAG 是什么" → [0.1, 0.2, ...]（向量A）
  "检索增强生成" → [0.1, 0.2, ...]（向量B）
  相似度 = cosine(A, B) ← 间接比较

CrossEncoder（cross-encoder）:
  "RAG 是什么" + "检索增强生成" → 模型 → 0.95 ← 直接比较
```

CrossEncoder 更准，但每个 query-doc 对都要单独计算，无法预存。

### CrossEncoder 重排序代码

```python
# src/retriever.py（部分）

class CrossEncoderReranker:
    """
    CrossEncoder 重排序，支持 4 种模式。
    """

    def __init__(
        self,
        api_base: str = "",       # API 模式：POST 到 {base}/rerank
        api_key: str = "not-needed",
        model_path: str = "",     # 本地路径 或 HuggingFace 模型 ID
    ):
        self.api_base = api_base.rstrip("/") if api_base else ""
        self.api_key = api_key
        self.model_path = model_path
        self._model = None
        self._load_model()

    def _load_model(self):
        """根据配置加载对应的模型。"""
        if self.api_base:
            # 模式1: API 调用
            self._mode = "api"
        elif self.model_path and Path(self.model_path).exists():
            # 模式2: 本地路径
            self._mode = "local"
            from sentence_transformers import CrossEncoder
            self._model = CrossEncoder(self.model_path)
        elif self.model_path:
            # 模式3: HuggingFace 自动下载
            self._mode = "huggingface"
            from sentence_transformers import CrossEncoder
            self._model = CrossEncoder(self.model_path)
        else:
            # 模式4: 禁用
            self._mode = "disabled"

    def rerank(
        self,
        query: str,
        documents: list[str],
        top_k: int = 5,
    ) -> list[tuple[int, float]]:
        """
        返回: [(doc_index_in_original_list, score), ...]
        score 越高越相关。
        """
        if self._mode == "disabled":
            # 禁用时返回原始顺序
            return list(enumerate([1.0] * len(documents)))[:top_k]

        if self._mode == "api":
            import httpx
            payload = {"query": query, "texts": documents}
            headers = {"Authorization": f"Bearer {self.api_key}"} if self.api_key != "not-needed" else {}

            response = httpx.post(f"{self.api_base}/rerank", json=payload, headers=headers, timeout=30)
            response.raise_for_status()
            result = response.json()

            # 解析返回格式（API 不同格式不同，这里假设 {results: [{index, score}]}）
            return [(r["index"], r["score"]) for r in result["results"]][:top_k]

        else:
            # local / huggingface 模式
            pairs = [[query, doc] for doc in documents]
            scores = self._model.predict(pairs)

            if isinstance(scores[0], (list, tuple)):
                scores = [s[0] for s in scores]

            ranked = sorted(enumerate(scores), key=lambda x: x[1], reverse=True)
            return [(idx, float(score)) for idx, score in ranked[:top_k]]
```

### 4 种 Reranking 模式总结

| 模式 | 配置 | 适用场景 |
|------|------|---------|
| **API** | `rerank_api_base` + `rerank_api_key` | 有专门的 Reranking 服务（如 Cohere） |
| **本地** | `rerank_model` = 本地目录路径 | 有本地 CrossEncoder 模型文件 |
| **HuggingFace** | `rerank_model` = 模型 ID | 愿意自动下载模型 |
| **禁用** | 留空 | 不想用 reranking，直接用 embedding 检索结果 |

## 6.5 完整混合检索流程

```python
class HybridRetriever:
    """
    完整检索流程：
    1. 向量检索（top_k candidates）
    2. BM25 检索（top_k candidates）
    3. RRF 融合
    4. 可选：CrossEncoder 重排序
    5. 可选：MMR 多样性
    """

    def __init__(
        self,
        vectorstore: VectorStore,
        embedder,
        hybrid_search: bool = True,
        dense_top_k: int = 20,
        sparse_top_k: int = 20,
        rrf_k: int = 60,
        rerank: bool = False,
        rerank_top_k: int = 20,
        mmr_enabled: bool = False,
        mmr_lambda: float = 0.3,
    ):
        self.vectorstore = vectorstore
        self.embedder = embedder
        # ... 保存参数

    def retrieve(
        self,
        query: str,
        top_k: int = 5,
        filter_metadata: dict = None,
    ) -> list[dict]:
        # 1. 问题向量化
        query_embedding = self.embedder.embed_single(query)

        # 2. 向量检索
        vector_results = self.vectorstore.query(
            query_embedding=query_embedding,
            n_results=self.dense_top_k,
            where_filter=filter_metadata,
        )

        # 3. BM25 检索
        if self.hybrid_search:
            bm25_results = self.bm25_retriever.retrieve(query, self.sparse_top_k)
            # RRF 融合
            fused = self._rrf_fusion(vector_results, bm25_results, self.rrf_k)
            candidates = fused[:self.rerank_top_k or top_k]
        else:
            candidates = vector_results

        # 4. 重排序（可选）
        if self.rerank and self.reranker:
            doc_texts = [c["text"] for c in candidates]
            reranked = self.reranker.rerank(query, doc_texts, top_k=top_k)
            candidates = [candidates[i] for i, _ in reranked]

        # 5. MMR 多样性（可选）
        if self.mmr_enabled:
            candidates = self._mmr_diversify(
                query_embedding, candidates, self.mmr_lambda, top_k
            )

        return candidates[:top_k]
```

## 完整检索参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `dense_top_k` | 20 | 第一阶段向量检索取多少个 |
| `sparse_top_k` | 20 | 第一阶段 BM25 取多少个 |
| `rrf_k` | 60 | RRF 融合参数，越大越偏 BM25 |
| `rerank` | false | 是否启用 CrossEncoder 重排序 |
| `rerank_top_k` | 20 | 重排序阶段送入 CrossEncoder 的候选数 |
| `mmr_enabled` | false | 是否启用 MMR 多样性 |
| `mmr_lambda` | 0.3 | 0=纯相关，1=纯多样 |
