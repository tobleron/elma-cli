# 第七章：生成阶段（LLM）

## 目标

把用户的问题 + 检索到的文档片段组合成 prompt，发给 LLM，生成最终回答。

## 核心思想

LLM 本身不知道你文档里的内容。通过 RAG，我们把**最相关的文档片段**和问题一起发给 LLM：

```
系统：你是一个助手，基于提供的文档片段回答问题。
用户：请基于以下片段回答：RAG 的分块策略有哪些？

---
[片段1] RAG 有 5 种分块策略：固定大小、递归、结构、语义、LLM驱动...
[片段2] 固定大小分块按字符数切分...
[片段3] 语义分块按句子切分再合并...
---

问题：RAG 的分块策略有哪些？
```

这就是"检索增强生成"——RAG 中的 **G（Generation）**。

## 7.1 Prompt 构建

```python
def build_rag_prompt(question: str, context_docs: list[dict]) -> str:
    """
    将问题 + 检索到的文档片段构建成 LLM prompt。
    """
    # 1. 把文档片段格式化成带编号的上下文
    context_parts = []
    for i, doc in enumerate(context_docs, 1):
        source = doc.get("metadata", {}).get("source", "unknown")
        chunk_idx = doc.get("metadata", {}).get("chunk_index", 0)
        context_parts.append(
            f"[{i}]（来源: {source}，第 {chunk_idx} 块）\n{doc['text']}"
        )

    context_text = "\n\n---\n\n".join(context_parts)

    # 2. 构建完整 prompt
    prompt = f"""你是一个专业的助手。请基于提供的参考文档片段回答用户的问题。
如果参考文档中没有足够的信息来回答问题，请如实说明，不要编造。

## 参考文档
{context_text}

## 用户问题
{question}

## 回答要求
1. 准确基于参考文档回答，不要编造信息
2. 适当引用参考文档的编号，如 [1]、[2]
3. 如果不确定，明确说明

回答："""
    return prompt
```

## 7.2 LLM API 调用

```python
# src/llm_api.py

from openai import OpenAI
from typing import Optional, Iterator
import tenacity

class LLMAPIClient:
    """
    LLM API 客户端，支持流式输出。
    """

    def __init__(
        self,
        api_base: str = "http://localhost:1234/v1",
        api_key: str = "not-needed",
        model: str = "gpt-4",
        max_tokens: int = 2048,
        temperature: float = 0.7,
    ):
        self.client = OpenAI(base_url=api_base, api_key=api_key)
        self.model = model
        self.max_tokens = max_tokens
        self.temperature = temperature

    def generate(
        self,
        prompt: str,
        system_prompt: Optional[str] = None,
        stream: bool = False,
    ) -> str | Iterator[str]:
        """
        调用 LLM 生成回答。
        stream=True 时返回迭代器（流式输出）。
        """
        messages = []
        if system_prompt:
            messages.append({"role": "system", "content": system_prompt})
        messages.append({"role": "user", "content": prompt})

        common_kwargs = {
            "model": self.model,
            "messages": messages,
            "max_tokens": self.max_tokens,
            "temperature": self.temperature,
        }

        if stream:
            return self._stream_generate(**common_kwargs)
        else:
            return self._non_stream_generate(**common_kwargs)

    def _non_stream_generate(self, **kwargs) -> str:
        """普通（非流式）调用。"""
        response = self.client.chat.completions.create(**kwargs)
        return response.choices[0].message.content

    def _stream_generate(self, **kwargs) -> Iterator[str]:
        """流式调用，yield 每个 token。"""
        kwargs["stream"] = True
        response = self.client.chat.completions.create(**kwargs)

        for chunk in response:
            if chunk.choices and chunk.choices[0].delta.content:
                yield chunk.choices[0].delta.content
```

## 7.4 RAG Pipeline（完整流程编排）

把检索 + 生成串联起来：

```python
# src/pipeline.py

class RAGPipeline:
    """
    完整的 RAG 流程：
    检索 → 组装 prompt → LLM 生成 → 返回回答 + 来源
    """

    def __init__(
        self,
        retriever: HybridRetriever,
        llm: LLMAPIClient,
        embedder,
    ):
        self.retriever = retriever
        self.llm = llm
        self.embedder = embedder

    def query(
        self,
        question: str,
        top_k: int = 5,
        filter_metadata: dict = None,
        stream: bool = False,
    ) -> dict:
        """
        返回 {
            "answer": "...",
            "sources": [...],
            "question": "...",
        }
        """
        # 1. 检索相关文档
        retrieved_docs = self.retriever.retrieve(
            question,
            top_k=top_k,
            filter_metadata=filter_metadata,
        )

        if not retrieved_docs:
            return {
                "answer": "没有找到相关文档，请尝试其他问题或重新索引文档。",
                "sources": [],
                "question": question,
            }

        # 2. 构建 prompt
        context_docs = [
            {"text": doc["text"], "metadata": doc.get("metadata", {})}
            for doc in retrieved_docs
        ]
        prompt = build_rag_prompt(question, context_docs)

        # 3. 调用 LLM
        answer = self.llm.generate(prompt, stream=stream)

        return {
            "answer": answer,
            "sources": [
                {
                    "text": doc["text"][:200] + "...",
                    "source": doc.get("metadata", {}).get("source", "unknown"),
                }
                for doc in retrieved_docs
            ],
            "question": question,
        }
```

## 7.5 检索失败时的处理

RAG 有时会找不到任何相关文档（top_k 太小、文档库为空等）。需要有兜底策略：

```python
def query(self, question, ...):
    retrieved_docs = self.retriever.retrieve(...)

    if not retrieved_docs:
        # 策略1：直接告诉用户
        return {
            "answer": "抱歉，知识库为空，请先上传并索引文档。",
            "sources": [],
        }

    # 策略2：降低 top_k 重新检索
    if len(retrieved_docs) < 3:
        retrieved_docs = self.retriever.retrieve(question, top_k=top_k * 2)

    # 策略3：提示用户问题可能超出知识库范围
    if not retrieved_docs:
        return {
            "answer": "未找到相关信息。您可以尝试：1. 上传更多相关文档 2. 调整检索参数",
        }
```

## 7.6 流式输出（更好的用户体验）

LLM 生成回答可能需要 5-10 秒。非流式：等全部生成完才返回。流式：逐词返回，用户能"看到思考过程"。

```python
# Streamlit 中使用流式
if st.button("发送"):
    placeholder = st.empty()

    full_answer = ""
    for token in rag_pipeline.query(question, stream=True):
        full_answer += token
        placeholder.markdown(full_answer + "▌")  # 光标闪烁效果

    placeholder.markdown(full_answer)  # 去掉光标
```
