# 第三章：分块策略（Chunking）

## 为什么需要分块？

LLM 和 Embedding 模型都有**上下文长度限制**：

- 把整本书直接发给 LLM → 超出 token 限制
- 把整本书向量化 → embedding 模型一般限制 512 tokens，太长效果差

所以需要先把文档切成小块（chunk），每个 chunk 单独向量化、独立检索。

## 分块的三个核心参数

```
chunk_size    = 每个块的最大字符数（或 token 数）
chunk_overlap = 相邻块之间的重叠字符数
min_chunk_size = 最小块大小（太短的块没意义）
```

```
文本: "The RAG pipeline has 5 stages. Each stage processes the data differently."

chunk_size=30, overlap=10:

块1: "The RAG pipeline has 5 stages."
块2 "         Each stage processes"  ← 重叠10字符
块3: "the data differently."
```

重叠的作用是**避免语义被切断** — 如果断开点在句号中间，会丢失上下文。

## 五种分块策略

### 3.1 固定大小分块（Fixed Size Chunking）

最简单粗暴，按字符数直接切。

```python
# src/chunkers/fixed_size_chunker.py

class FixedSizeChunker:
    """最简单的分块器：按固定字符数切分。"""

    def __init__(self, chunk_size: int = 512, overlap: int = 50):
        self.chunk_size = chunk_size
        self.overlap = overlap

    def chunk(self, text: str) -> list[dict]:
        """
        返回格式: [{"text": "...", "metadata": {...}}, ...]
        """
        if not text or not text.strip():
            return []

        chunks = []
        start = 0
        text_len = len(text)

        while start < text_len:
            end = start + self.chunk_size

            # 尽量在句子边界断开
            if end < text_len:
                # 在 chunk_size 范围内找最后一个句号/换行
                cut = text.rfind("。", start, end)
                if cut > start:
                    end = cut + 1
                else:
                    # 没找到句号，找换行
                    cut = text.rfind("\n", start, end)
                    if cut > start:
                        end = cut

            chunk_text = text[start:end].strip()
            if len(chunk_text) >= 20:  # 过滤掉太短的块
                chunks.append({
                    "text": chunk_text,
                    "metadata": {
                        "start_char": start,
                        "end_char": end,
                        "chunk_size": len(chunk_text),
                    }
                })

            start = end - self.overlap  # 重叠滑动

        return chunks
```

**优点**：简单、稳定、可复现
**缺点**：不考虑语义，经常在句子中间断开

### 3.2 递归分块（Recursive Chunking）

固定大小的问题是"不考虑语义"。递归分块则按**优先级递减的分隔符**不断尝试切分：

```
优先级 1: "\n\n"（段落）     → 先按段落切
    ↓ 如果段落太长
优先级 2: "\n"（换行）      → 按句子/短句切
    ↓ 如果还是太长
优先级 3: ". "（句号+空格） → 按句子切
    ↓ ...
优先级 4: " "（空格）       → 逐词切（最后手段）
```

```python
# src/chunkers/recursive_chunker.py

class RecursiveChunker:
    def __init__(
        self,
        chunk_size: int = 512,
        overlap: int = 50,
        separators: list[str] = None,
        min_chunk_size: int = 50,
    ):
        self.chunk_size = chunk_size
        self.overlap = overlap
        self.min_chunk_size = min_chunk_size
        # 按优先级排序的分隔符
        self.separators = separators or ["\n\n", "\n", ". ", ", ", " "]

    def chunk(self, text: str) -> list[dict]:
        return self._split(text, self.separators)

    def _split(self, text: str, separators: list[str]) -> list[dict]:
        """
        递归地把 text 按分隔符逐级切分，直到每块 <= chunk_size。
        """
        if not text or not text.strip():
            return []

        # 最高优先级的分隔符
        sep = separators[0]
        remaining = separators[1:]  # 次高级别备用

        parts = text.split(sep)
        result = []

        for part in parts:
            part = part.strip()
            if not part:
                continue

            # 如果这块已经够小，直接保留
            if len(part) <= self.chunk_size:
                result.append(part)
            else:
                # 否则用次低优先级分隔符继续切
                if remaining:
                    sub_chunks = self._split(part, remaining)
                    result.extend(sub_chunks)
                else:
                    # 没有更低的分隔符了，强制截断（不应该发生）
                    result.append(part[:self.chunk_size])

        # 把相邻小块合并（避免太多碎片）
        merged = self._merge_chunks(result)
        return merged

    def _merge_chunks(self, chunks: list[str]) -> list[dict]:
        """把相邻小块合并成不超过 chunk_size 的大块。"""
        if not chunks:
            return []

        merged_chunks = []
        current = ""
        current_size = 0

        for chunk in chunks:
            chunk_len = len(chunk)

            if current_size + chunk_len + 1 <= self.chunk_size:
                # 能合并就合并，用空格分隔
                current = (current + " " + chunk).strip() if current else chunk
                current_size = len(current)
            else:
                # 当前块已满，保存，然后开启新块
                if current:
                    merged_chunks.append({"text": current, "metadata": {}})
                # 如果单个 chunk 就超长了，强制截断
                if chunk_len > self.chunk_size:
                    for i in range(0, chunk_len, self.chunk_size - self.overlap):
                        sub = chunk[i:i + self.chunk_size]
                        if len(sub) >= self.min_chunk_size:
                            merged_chunks.append({"text": sub, "metadata": {}})
                            if i + self.chunk_size < chunk_len:
                                i -= self.overlap
                else:
                    current = chunk
                    current_size = chunk_len

        if current:
            merged_chunks.append({"text": current, "metadata": {}})

        return merged_chunks
```

**优点**：比固定大小更"语义完整"
**缺点**：代码复杂，合并逻辑容易出 bug

### 3.3 结构分块（Structure Chunking）

针对 Markdown/HTML 这类有结构的文档，按**标题层级**切分：

```markdown
# 第一章 RAG 概述

RAG 是检索增强生成...

## 1.1 什么是 RAG

RAG 由三部分组成...
```

这种文档中，`#` 标题 + 后续内容是一个天然的"语义单元"，不应该被截断。

```python
# src/chunkers/structure_chunker.py

import re

class StructureChunker:
    """按文档结构（标题、段落）切分，保留逻辑单元完整性。"""

    def __init__(
        self,
        chunk_size: int = 512,
        overlap: int = 50,
        split_on: str = "paragraphs",  # headings | paragraphs | lists | code_blocks
    ):
        self.chunk_size = chunk_size
        self.overlap = overlap
        self.split_on = split_on

    def chunk(self, text: str) -> list[dict]:
        """返回 [{"text": "标题\\n内容", "metadata": {"heading": "..."}}, ...]"""
        if not text:
            return []

        # 1. 识别标题行（# Markdown 格式）
        heading_pattern = re.compile(r"^(#{1,6})\s+(.+)$", re.MULTILINE)
        headings = [(m.start(), m.group(1), m.group(2)) for m in heading_pattern.finditer(text)]

        # 2. 如果没有标题，退化为段落分块
        if not headings:
            return self._chunk_by_paragraphs(text)

        chunks = []
        for i, (heading_pos, level, heading_text) in enumerate(headings):
            # 确定这个标题的范围（到下一个同级/上级标题之前）
            end = headings[i + 1][0] if i + 1 < len(headings) else len(text)

            section_text = text[heading_pos:end].strip()
            chunk_text = f"{heading_text}\n\n{section_text}"

            if len(chunk_text) >= 50:
                chunks.append({
                    "text": chunk_text,
                    "metadata": {
                        "heading": heading_text,
                        "heading_level": len(level),
                    }
                })

        return chunks

    def _chunk_by_paragraphs(self, text: str) -> list[dict]:
        """没有标题时，按段落分块。"""
        paragraphs = [p.strip() for p in text.split("\n\n") if p.strip()]
        return [
            {"text": p, "metadata": {}}
            for p in paragraphs if len(p) >= 50
        ]
```

**优点**：完美保留 Markdown/标题结构
**缺点**：纯 TXT 文档没有结构，效果差

### 3.4 语义分块（Semantic Chunking）

核心思想：**把语义相近的句子放在一起**。

流程：
```
1. 把文本按句子拆开
2. 每个句子独立生成 embedding 向量
3. 相邻句子如果 embedding 相似度高 → 合并成一个块
4. 最后按 chunk_size 上限截断
```

```python
# src/chunkers/semantic_chunker.py

class SemanticChunker:
    """语义分块：按句子切分 → 向量化 → 按相似度合并。"""

    def __init__(
        self,
        chunk_size: int = 512,
        overlap: int = 50,
        similarity_threshold: float = 0.7,
        min_chunk_size: int = 100,
        embedder=None,  # EmbedderAPIClient 实例
    ):
        self.chunk_size = chunk_size
        self.overlap = overlap
        self.similarity_threshold = similarity_threshold
        self.min_chunk_size = min_chunk_size
        self.embedder = embedder

    def chunk(self, text: str) -> list[dict]:
        # 1. 句子分割
        sentences = self._split_into_sentences(text)
        if len(sentences) <= 1:
            return [{"text": text, "metadata": {}}]

        # 2. 向量化（需要 embedder）
        if self.embedder is None:
            raise ValueError("SemanticChunker 需要 embedder 来计算句子相似度")
        embeddings = self.embedder.embed_batch(sentences)

        # 3. 按相似度合并相邻句子
        chunks = []
        current_chunk = sentences[0]
        current_embedding = embeddings[0]

        for i in range(1, len(sentences)):
            sim = cosine_similarity(current_embedding, embeddings[i])

            if sim >= self.similarity_threshold and len(current_chunk) + len(sentences[i]) <= self.chunk_size:
                # 相似且没超限 → 合并
                current_chunk += " " + sentences[i]
            else:
                # 不相似或超限 → 保存当前块，开启新块
                if len(current_chunk) >= self.min_chunk_size:
                    chunks.append({"text": current_chunk, "metadata": {}})
                current_chunk = sentences[i]
                current_embedding = embeddings[i]

        # 最后一个块
        if len(current_chunk) >= self.min_chunk_size:
            chunks.append({"text": current_chunk, "metadata": {}})

        return chunks

    def _split_into_sentences(self, text: str) -> list[str]:
        """简单的句子分割（用句号、问号、感叹号）。"""
        import re
        sentences = re.split(r"(?<=[。！？.!?])\s+", text)
        return [s.strip() for s in sentences if s.strip()]


def cosine_similarity(a: list[float], b: list[float]) -> float:
    """计算两个向量的余弦相似度。"""
    import math
    dot = sum(x * y for x, y in zip(a, b))
    norm_a = math.sqrt(sum(x * x for x in a))
    norm_b = math.sqrt(sum(x * x for x in b))
    return dot / (norm_a * norm_b + 1e-10)
```

**优点**：语义最完整
**缺点**：需要调用 Embedding API（成本+延迟），无法离线工作

### 3.5 LLM 分块（LLM-based Chunking）

最精确但最贵：用 LLM 自行判断"这段文字的语义边界在哪里"。

```python
# src/chunkers/llm_chunker.py

class LLMChunker:
    """由 LLM 判断语义完整的分块边界。"""

    def __init__(
        self,
        chunk_size: int = 512,
        overlap: int = 50,
        max_llm_chunk_size: int = 200,  # 每次发给 LLM 的最大字符数
        llm_client=None,  # LLMAPIClient 实例
    ):
        self.chunk_size = chunk_size
        self.overlap = overlap
        self.max_llm_chunk_size = max_llm_chunk_size
        self.llm_client = llm_client

    def chunk(self, text: str) -> list[dict]:
        if not self.llm_client:
            raise ValueError("LLMChunker 需要 llm_client")

        # 1. 先粗切（固定大小），每块发给 LLM 判断边界
       粗块s = self._rough_split(text)

        chunks = []
        for part in 粗块s:
            # 2. 让 LLM 判断这个粗块应该怎么细分
            boundaries = self._llm_find_boundaries(part)
            # 3. 按边界切分
            for (start, end) in boundaries:
                chunk_text = part[start:end].strip()
                if len(chunk_text) >= 50:
                    chunks.append({"text": chunk_text, "metadata": {"llm_guided": True}})

        return chunks

    def _llm_find_boundaries(self, text: str) -> list[tuple[int, int]]:
        """调用 LLM 获取语义边界。"""
        prompt = f"""请分析以下文本，找出语义完整的句子边界。
返回 JSON 格式：{{"boundaries": [[0, 45], [45, 120], ...]}}
每个 [start, end] 表示一个语义完整片段的字符位置。

文本：
{text[:1000]}
"""
        response = self.llm_client.generate(prompt)
        import json
        result = json.loads(response)
        return [tuple(b) for b in result["boundaries"]]
```

**优点**：最准确，语义边界判断最合理
**缺点**：需要 LLM API 调用，成本高、速度慢

## 五种策略对比

| 策略 | 速度 | 语义完整度 | 离线可用 | 适用场景 |
|------|------|-----------|---------|---------|
| 固定大小 | ⚡⚡⚡⚡⚡ | ⭐ | ✅ | 快速原型、不在乎语义的场景 |
| 递归 | ⚡⚡⚡ | ⭐⭐⭐ | ✅ | 通用场景，最常用 |
| 结构 | ⚡⚡⚡ | ⭐⭐⭐⭐ | ✅ | Markdown / HTML 文档 |
| 语义 | ⚡⚡ | ⭐⭐⭐⭐⭐ | ❌（需 embedder） | 质量要求高的场景 |
| LLM | ⚡ | ⭐⭐⭐⭐⭐ | ❌（需 LLM） | 最高质量需求，不计成本 |

## 分块大小的经验法则

| Embedding 模型 | 推荐 chunk_size | 说明 |
|---------------|----------------|------|
| OpenAI `text-embedding-3-small` | 256~512 tokens | 短块更精确 |
| BGE-large | 512 tokens | 中等长度 |
| 本地模型（如 all-MiniLM） | 256~384 tokens | 短块效果好 |

> 注意：代码中我们用**字符数**，实际应该是 **token 数**。粗略估算：1 token ≈ 4 个中文字符，或 0.75 个英文单词。
